# IRIDIA Pre-AFib Episode Analysis (ML Focus)

> Generated on 2026-03-21 11:47:36

## Objective

Evaluate RR data quality before each AF episode at horizons of `10, 20, 30, 60, 120, 180` minutes to determine suitability for machine learning.

## Metrics Checklist for ML Readiness

| Metric | Definition | ML Threshold | Why it matters |
|---|---|---|---|
| `is_sufficient_duration` | Segment contains at least 95% of requested pre-AFib horizon duration. | `True` | Insufficient context increases label noise and feature instability. |
| `rr_count` | Number of RR intervals in segment. | `>= 200` | Low counts produce weak HRV and distribution estimates. |
| `outlier_count_total_phys / outlier_pct_phys` | RR values outside [250, 2500] ms. | `<= 5%` | High outlier rate indicates artifact or poor signal quality. |
| `duplicate_pct` | Fraction of consecutive RR differences equal to zero. | `<= 10%` | Excess duplicates can indicate quantization or sensor artifacts. |
| `std_rr_ms` | RR standard deviation. | `10-400 ms (soft range)` | Near-zero or extreme variability can be non-physiologic. |
| `quality_score` | Composite score from coverage, count, outliers, duplicates and variability. | `>= 70` | Single summary metric for model-ready segment filtering. |

## Data and Method

- Each AF episode uses its RR start index as AF onset.
- For each horizon, RR intervals immediately preceding AF onset are extracted.
- Features include RR central tendency, variability, signal quality, and artifacts.
- A heuristic quality score (0-100) and `ml_ready` flag are computed per episode/horizon.

## Statistical Evidence (Outliers and Quality)

- Physiological outlier is defined as RR < **250 ms** or RR > **2500 ms**.
- For each segment, both **outlier counts** and **outlier percentages** are reported.
- IQR-based outliers are additionally reported to detect distribution-level anomalies within each segment.
- Report includes P95 outlier percentage to highlight worst-case tails, not only averages.

- **10 min**: mean physical outliers = **2.1** (0.22%), P95=1.14%, mean IQR-outliers=7.39%.
- **20 min**: mean physical outliers = **3.7** (0.20%), P95=1.03%, mean IQR-outliers=7.32%.
- **30 min**: mean physical outliers = **5.6** (0.21%), P95=0.87%, mean IQR-outliers=7.75%.

## Key Findings

- Best-performing horizon by ML readiness is **10 minutes** (59.79% episodes marked ML-ready).
- Shorter horizons usually have higher coverage; longer horizons can add context but may lose episodes due to insufficient pre-AFib history.
- For model design, use horizons with both high `ml_ready_pct` and stable RR count distributions.
- Outlier evidence supports that many segments are usable, but strict filtering is still required to remove high-artifact segments before model training.

## Recommended ML Filtering Rules

- `is_sufficient_duration == True`
- `rr_count >= 200`
- `phys_out_of_range_pct <= 5`
- `duplicate_pct <= 10`
- `quality_score >= 70`
